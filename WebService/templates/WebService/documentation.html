WS-VulnS est muni d’une interface graphique facilitant l’utilisation de l’outil alliant convivialité et simplicité mais il est aussi à noter que le but de notre outil est la détection de vulnérabilités dans les Web Services avant leur déploiement, on suppose donc que notre produit sera utilisé par des personnes expérimentées tel qu’un administrateur qui aura toutes les compétences nécessaires pour comprendre et analyser les résultats obtenus. De plus, il sera capable de configurer l’outil selon les résultats voulus, c’est pour cela que nous avons étendu la version graphique à une autre version en ligne de commande, plus avancée, permettant à l’utilisateur d’effectuer des tests plus profonds et surtout personnalisés dans le sens où il peut lui-même fixer les différents paramètres intervenant dans la détection des attaques sélectionnées tels que le choix des algorithmes de Clustering parmi ceux proposés, etc.

Nous présentons sous forme de captures d’écran un cas d’utilisation de WS-VulnS. L’utilisateur va choisir un Web Service de type SOAP, il va ensuite effectuer quelques tests fonctionnels sur ce dernier et passer aux tests de vulnérabilités en choississant l’opération et les attaques à tester, ces dernières seront configurées par l’utilisateur. Enfin, après avoir lancé la détection, il récupérera les résultats grâce à la fonctionnalité de génération de rapport.

Dans le cas d’un utilisateur plus expérimenté voulant pousser encore plus les tests sur le Web Service en question, celui-ci peut utiliser la version avancée de WS-VulnS en ligne de commande, permettant de configurer de manière très flexible la détection de vulnérabilités à travers un fichier de configuration très simple à modifier selon les choix de l’utilisateur.


Le fichier de configuration est composé de cinq sections, une première section contenant des valeurs par défaut données à certains paramètres dans le cas où ces derniers ne sont pas définis par l’utilisateur. Les quatre autres sections contiennent les paramètres de configuration regroupant le choix du type de Web Service ainsi que l’opération à tester, le choix des attaques à tester ainsi que le choix des valeurs des variables intervenant dans la détection du DoS et la détection des injections. Des directives sont données à l’utilisateur afin de l’orienter dans la modification du fichier de configuration.
